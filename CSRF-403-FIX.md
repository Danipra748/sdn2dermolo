# 403 Forbidden Error - FIXED

## Problem
The application was returning a 403 Forbidden error because AJAX/fetch requests were not including the CSRF token in their headers. Laravel's CSRF protection middleware was blocking these requests for security reasons.

## Root Cause
The SPA (Single Page Application) navigation system in `public/js/spa.js` was making fetch requests without including the CSRF token, causing Laravel to reject them with a 403 response.

## Solution Applied

### 1. Added CSRF Token Meta Tag
**File:** `resources/views/layouts/app.blade.php`

Added the CSRF token meta tag to the `<head>` section:
```html
<meta name="csrf-token" content="{{ csrf_token() }}">
```

This makes the CSRF token available to all JavaScript code on the page.

### 2. Updated SPA Fetch Requests
**File:** `public/js/spa.js`

Modified the `fetchContent()` function to include the CSRF token in request headers:
```javascript
// Get CSRF token from meta tag
const csrfToken = document.querySelector('meta[name="csrf-token"]')?.getAttribute('content') || '';

return fetch(cacheBustUrl, {
    headers: {
        Accept: 'application/json',
        'X-Requested-With': 'XMLHttpRequest',
        'X-CSRF-TOKEN': csrfToken,  // Added this line
    },
    cache: isNoCache ? 'no-store' : 'default',
});
```

### 3. Updated Axios Configuration
**File:** `resources/js/bootstrap.js`

Updated the Axios setup to automatically include CSRF token in all requests:
```javascript
// Setup CSRF token for all Axios requests
const csrfToken = document.querySelector('meta[name="csrf-token"]')?.getAttribute('content');

if (csrfToken) {
    window.axios.defaults.headers.common['X-CSRF-TOKEN'] = csrfToken;
} else {
    console.warn('[Axios] CSRF token meta tag not found');
}
```

## Testing
After implementing these changes:
1. All SPA navigation (home, fasilitas, guru, prestasi, galeri, tentang, program, berita) should work without 403 errors
2. All AJAX requests throughout the application will include the CSRF token automatically
3. The application remains secure against CSRF attacks

## Files Modified
- `resources/views/layouts/app.blade.php` - Added CSRF meta tag
- `public/js/spa.js` - Updated fetch requests to include CSRF token
- `resources/js/bootstrap.js` - Updated Axios CSRF setup

## Notes
- The CSRF token is automatically generated by Laravel on each page load
- The token is tied to the user's session and changes periodically for security
- This fix maintains Laravel's built-in CSRF protection while allowing legitimate AJAX requests
